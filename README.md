# TOAST — прошивка «скейл-клавиатура»

Кастомная прошивка для MIDI-контроллера **SpaceMelodyLab TOAST v1.1**:
8 кнопок превращаются в клавиатуру с музыкальными скейлами, 8 крутилок —
в назначаемые CC-контроллеры и селекторы нот.

## Возможности

- **8 игровых кнопок** с полной полифонией — аккорды играются
- **Широкие аккорды из скейла**: удержание октавной кнопки превращает
  игровые кнопки в большие войсинги, раскинутые на 2–2.5 октавы (8 maj9 +
  8 квартовых), все ноты из текущего скейла — любая последовательность
  звучит слитно и «кинематографично»
- **76 скейлов** (мажор, миноры, лады, пентатоники, блюз, японские и др.)
  с перемещаемой тоникой
- **Своя нота на каждой кнопке**: парная крутилка пробегает по ступеням
  выбранного скейла через весь MIDI-диапазон
- **Октавный сдвиг ±3**
- **8 потов → MIDI CC** (по умолчанию CC1–CC8, номера меняются с панели)
- **Velocity нот** крутится девятым потом прямо во время игры
- **Два слоя потов**: по умолчанию поты выбирают ноты кнопок, а CC — под
  шифтом; комбо меняет их местами (переключение действует до перезагрузки,
  прибор всегда стартует в нот-слое)
- **MIDI-канал 1–16** выбирается с панели
- События идут одновременно в **USB-MIDI и DIN/TRS MIDI OUT**
- **Штатная радиопанель** с двумя кнопками принимается запаянным nRF24L01+;
  проверенные кнопки по умолчанию дублируют первые две игровые ноты
- Настройки сохраняются в **EEPROM отложенно**, без износа от каждого
  прохода loop и без изменения первых 10 служебных байт
- **OLED**: пиано-октава с нажатыми нотами и тоникой, статус,
  подсказки действий; светодиод мигает при отправке

При первом включении или невалидном EEPROM: хроматический скейл, канал 1,
первая кнопка = C3 (C3 = MIDI 60), поты шлют CC1–CC8. После первого
сохранения при старте восстанавливаются пользовательские настройки.

## Управление

Игровые кнопки слева направо: KEY6 → KEY10 → KEY5 → KEY9 → KEY8 →
KEY7 → KEY3 → KEY2 (нижний/средний ряд зигзагом).

| Действие | Результат |
|---|---|
| 8 игровых кнопок | ноты |
| KEY11 / KEY1 (короткий тап) | октава вниз / вверх |
| **держать KEY11** + игровая кнопка | аккорд Set A (раскрытый maj9, ~2 октавы) |
| **держать KEY1** + игровая кнопка | аккорд Set B (квартовый стек, ~2.5 октавы) |
| POT1–8 | MIDI CC |
| POT9 (value) | velocity нот 1–127 |
| **SHIFT (KEY4)** + POT9 | выбор скейла |
| SHIFT + POT1–8 | нота закреплённой кнопки |
| SHIFT + KEY11 + POT9 | MIDI-канал |
| SHIFT + KEY11 + POT1–8 | CC-номер этого пота |
| SHIFT + KEY1 + POT9 | яркость OLED |
| SHIFT + KEY1 + POT1 | тоника скейла (C…B) |
| SHIFT + KEY11 + KEY1 вместе | смена слоёв потов (CC ↔ ноты), флажок «NOTES» |
| SHIFT + первая игровая кнопка | сброс нот кнопок к ступеням скейла |

Пары кнопка-пот: KEY6-POT5, KEY10-POT1, KEY5-POT6, KEY9-POT2,
KEY8-POT7, KEY7-POT3, KEY3-POT8, KEY2-POT4.

Выбираемая крутилкой нота подсвечивается на пиано-клавиатуре вместе
с текстовой подсказкой («BTN3 = D#4»). После смены режима пот
«просыпается» только после реального поворота — случайных скачков
CC и настроек не бывает.

Скейлы (76): Chromatic, Major, Minor, Harmonic/Melodic Minor, Dorian,
Phrygian, Lydian, Mixolydian, Locrian, Pentatonic Maj/Min, Blues
Min/Maj, Whole Tone, Diminished W-H/H-W, Phrygian Dominant, Hungarian
Minor/Major, Double Harmonic, Hirajoshi, In Sen, Iwato, Yo, Harmonic
Major, Lydian Dominant, Altered, Locrian #2, Mixolydian b6, Dorian b2,
Bebop Dominant/Major, Neapolitan Minor/Major, Enigmatic, Prometheus,
Augmented, Tritone, Persian, Romanian, Egyptian, Kumoi, Pelog,
Messiaen modes 3–7, Bohlen-Pierce Lambda и Chromatic (неоктавные,
период — тритава из 19 полутонов), Harmonics (обертоновый ряд 8–16),
Tcherepnin, плюс блюзовый набор (Minor maj7, Heptatonic, Dorian hex,
Modified, Octatonic, Phrygian), Chromatic Dorian/Phrygian, Ultra
Locrian, Algerian, Oriental, Enigmatic 8/Minor, Bhairav, Ritsu,
Puravi bVI, Nohkan, Flamenco, Bhairubahar, Adonai Malakh, Zirafkend,
Utility Minor.

Смена скейла/тоники **сохраняет** назначения кнопок: каждая кнопка держит
свою ступень-позицию и просто озвучивается по новому скейлу (позиции вне
диапазона подрезаются). Полный сброс раскладки к дефолту — отдельным комбо
SHIFT + первая игровая кнопка («NOTES RESET»). Октавный сдвиг сохраняется.

### Аккорды

Октавные кнопки теперь срабатывают на **отпускание** (короткий тап =
сдвиг октавы), а **удержание** служит модификатором аккордов:

- удерживай **KEY11 (октава вниз)** и нажимай игровые кнопки → **Set A**
  (по умолчанию раскрытый maj9/min9, голоса разнесены на ~2 октавы —
  большой «кинематографичный» аккорд из одной кнопки);
- удерживай **KEY1 (октава вверх)** → **Set B** (квартовый стек на ~2.5
  октавы, открытый эпичный модальный звук).

Каждая из 8 игровых кнопок даёт свой широкий аккорд, построенный от её
ноты, так что получается 8 + 8 аккордов. Все голоса берутся **из текущего
скейла** —
поэтому аккорды всегда в тональности и любую их последовательность можно
играть подряд без фальши. Смени скейл или тонику — и весь набор аккордов
меняет характер (например, MAJOR даёт Cmaj7/Dm7/Em7…, экзотические скейлы —
экзотические созвучия). На хроматическом скейле по умолчанию Set A звучит
цельнотоново; для классических аккордов выбери диатонический скейл.

Смещения голосов задаются в `config.h` (`CHORD_SET_A` / `CHORD_SET_B`
в ступенях скейла, до `CHORD_VOICES` нот) — правятся одной строкой.

Скейл, тоника, октава, MIDI-канал, velocity, яркость OLED, CC-номера и
назначения нот сохраняются спустя 2 секунды после последнего изменения.
Слой потов не сохраняется — прибор всегда стартует в нот-слое.
Запись идёт через `EEPROM.update()` по одному байту за цикл и ставится на
паузу, пока удерживается нота. Собственный блок начинается с EEPROM byte 10;
bytes 0–9 не используются, а штатные ID радиопанелей в `0x0208..0x0213`
только читаются. Поэтому возврат к официальной прошивке и её pairing
остаются возможны. Но custom block занимает bytes 10–48, где в дампе
официальной EEPROM тоже были параметры; для побитового возврата именно к
прежним official-настройкам надо восстановить сохранённый EEPROM backup.
CRC не позволит загрузить полу-записанный блок, но хранилище однослотовое:
если выключить питание ровно во время короткой записи, следующий старт
безопасно возьмёт defaults, и пользовательские настройки придётся задать ещё раз.

## Беспроводные кнопки

У установленного модуля nRF24L01+ собственный протокол, это не BLE.
Подтверждённые параметры оригинала: CE=D10, CSN=D9, SPI=D14/D15/D16,
канал 120 (2520 MHz), 250 kbit/s, CRC16, auto-ack, pipe 1 с адресом
`baLMS`, статический payload 7 bytes. Живой пульт передаёт ID `IlJRf4`:
code 0 даёт `0x40/0x00` на press/release, code 1 — `0x41/0x01`.

Первая панель/code 0 и code 1 по умолчанию играют ноты игровых кнопок 1
и 2. Вторая предусмотренная штатной EEPROM панель заранее получает места
3 и 4. Таблица `NRF_BUTTONS[]` в `config.h` подробно прокомментирована:
любое место можно отключить либо заменить на momentary MIDI CC с отдельными
значениями нажатия и отпускания. Pairing новые ID эта прошивка намеренно не
делает — его безопаснее выполнить официальной прошивкой один раз.
Если пульт потеряет питание между PRESS и RELEASE, 30-секундная страховка
сама отпустит зависшую radio note/CC; лимит меняется в `config.h`.

## Железо

Плата TOAST v1.1: Arduino Pro Micro (ATmega32U4, 5V/16MHz), 3× 74HC4051,
9 потенциометров, 11 кнопок, OLED SSD1306 128×32 (I²C), MIDI OUT на
джеке 3.5мм (TRS), светодиод и запаянный nRF24L01+-совместимый модуль.
Распиновка платы уже вписана в
[`src/toast_firmware/config.h`](src/toast_firmware/config.h).

## Сборка и прошивка

```bash
pio run -e toast_firmware -t upload
```

Плата шьётся через штатный бутлоадер (Caterina). Если загрузка не
стартует — дважды быстро замкните RST на GND и повторите upload в
течение ~8 секунд.

Поддерживаемая и проверенная сборка — PlatformIO. В Arduino IDE библиотека
Adafruit SSD1306 по умолчанию добавляет две splash-картинки, и этот полный
вариант уже не помещается во flash. Если Arduino IDE всё-таки необходима,
сначала глобально задайте для сборки библиотеки `SSD1306_NO_SPLASH`, затем
откройте `src/toast_firmware/toast_firmware.ino`, выберите «Arduino Micro»
и установите MIDIUSB, Adafruit SSD1306 и Adafruit GFX.

## Настройка под себя

Всё в [`config.h`](src/toast_firmware/config.h): роли кнопок, пары
кнопка-пот, стартовые CC-номера, MIDI-канал, пороги дебаунса и
«пробуждения» потов, velocity, время подсказок и действия радиокнопок.

## MIDI-выход TRS: тип A или B?

Стандарт — тип A. Если DIN-адаптер типа A не заработает, попробуйте
адаптер типа B (перепутаны Tip и Ring).

## Планы

- UI для выбора Note/CC-действия радиокнопок без пересборки
- Опциональный pairing второй штатной панели с отдельным подтверждением

BLE MIDI через этот радиомодуль в планы не входит: nRF24L01+ не поддерживает
BLE, а имитация через advertising не дала бы нормальной доставки событий.
